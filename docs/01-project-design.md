# 방명록 프로젝트 설계

## 1. 서비스 설명

사용자가 자신의 글을 남기고 다른 사용자의 글을 확인할 수 있는 **간단한 방명록 서비스**입니다.

---

## 2. 전체 흐름

1. 사용자가 웹 페이지에서 방명록 글을 작성하거나 전체 방명록을 조회 요청합니다.
2. 프론트엔드는 입력 데이터를 백엔드 API(`guestbooks`)로 전송하거나, 전체 방명록 조회를 요청합니다.
3. 백엔드는 요청을 받아 데이터베이스에서 데이터를 읽거나 저장합니다.
4. 처리 결과(저장 성공 여부 또는 방명록 목록)를 프론트엔드로 반환합니다.
5. 프론트엔드는 반환받은 데이터를 화면에 표시하여 사용자가 확인할 수 있도록 합니다.

---

## 3. 프론트엔드 역할

- 사용자 인터페이스(UI) 제공: 방명록 작성 폼과 방명록 목록 표시
- 사용자 입력 검증: 글 작성 시 필수 항목(닉네임, 내용) 체크
- API 요청 및 응답 처리:
  - 작성된 글을 백엔드로 전송 (`POST /guestbooks`)
  - 전체 방명록 데이터를 요청하여 화면에 렌더링 (`GET /guestbooks`)
- 사용자 경험 개선: 글 작성 후 자동 갱신, 에러 메시지 표시 등

---

## 4. 백엔드 역할

- REST API 제공:
  - `POST /guestbooks` → 방명록 작성
  - `GET /guestbooks` → 전체 방명록 조회
- 데이터 처리 및 저장: 데이터베이스에 글 저장 및 조회
- 입력 데이터 검증 및 에러 처리
- 프론트엔드 요청에 대한 JSON 형식 응답 반환

---

## 5. 배포 구조 요약

| ------------------------ | ---------------------------------------------------------------------------------------------- |
